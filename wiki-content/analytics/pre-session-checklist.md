# Trigger

Run once per session, in order, before any order is placed — the full
integration of the entire "Qu'an Execution Playbook" into one sequence.

# Steps

1. **Session Close Reading** — record DIPLTR at CW+0.9/+1.0; count ZC flags and the final ZC's quadrant; assess entropy (clean/moderate/turbulent); read SOPG/SOPC Latent at CW+0.7/+0.8/+0.9; synthesize Strong/Moderate/No Prior. See [Deep Strike Analysis](/analytics/deep-strike-analysis.md) Part I.
2. **Observable Field Scan** — pull the prior close's Book sheet; apply the PDSL criteria (Mass, Kurt, LR, |A|); mark each candidate PDSL (4/4) or DSC (3/4).
3. **Gradient Classification** — classify each PDSL's Force gradient (ascending/descending/phase boundary); orient and calculate the Fibonacci quarter-level grid for each PDSL pair. See [Fibonacci Strike Architecture](/analytics/fibonacci-strike-architecture.md).
4. **TSC Overlay + Dealer Temporal Assessment** — overlay the Phase 1 prior onto each PDSL, noting alignment/conflict; check DR3, ICF Time Density, and Dealer Premium Time; mark each PDSL LIVE (pre-session-order-eligible) or SPENT (live-confirmation only).
5. **Deep Strike Scorecard** — score each PDSL 0–10; Tier 1 (8–10) gets full pre-session orders, Tier 2 (6–7) gets reduced size with no overnight orders, Tier 3 (<6) is watch-only.
6. **Conductance Gate** — read the pre-market TSC conductance classification; apply condFactor to every planned allocation; below 0.50, cancel all pre-session limits and go live-only.
7. **Order Architecture Construction** — build the three-layer structure (see [Three-Layer Execution Model](/analytics/three-layer-execution-model.md)) for each Tier 1 PDSL, with OCO brackets and an explicit written cancel condition per order; Tier 2 PDSLs get Layer A only, noted as requiring live μ-Wave confirmation for B/C.
8. **Final Status Check** — assess personal status (sleep, hydration, emotional baseline; degraded = cut all sizes 50%); re-read the scorecard and check whether conviction is genuine; write the single most important PDSL for the session; place orders.

# Failure modes

- Skipping straight to order construction without the close reading or conductance gate is exactly the discretionary behavior this checklist exists to prevent — every phase gates the next.
- A "genuine vs. forced" conviction check at Phase 8 is explicitly there to catch a scorecard that was unconsciously massaged to justify a trade already wanted.
